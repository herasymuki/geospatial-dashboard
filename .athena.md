## Build Fix — 2026-05-29 (Revision 00004-vzn)
- **Root cause**: `deck.gl` meta-package (v9.3.2) transitively pulls in `@deck.gl/react`
  which imports `react` — not installed in this Vue app. Rollup hard-errored on unresolved `react`.
- **Fix 1**: Removed `"deck.gl": "9.3.2"` from `package.json` (meta-package).
  All needed scoped packages (`@deck.gl/core`, `@deck.gl/layers`, `@deck.gl/geo-layers`,
  `@deck.gl/aggregation-layers`) are already listed explicitly.
- **Fix 2**: Added `external: ["react", "react-dom"]` to `vite.config.js` rollupOptions
  as a safety net for any other transitive React references.
- **Commit**: 0cfddfd7002af157c619bd413c6b14740569a3a2
- **Revision**: global-conflicts-athena-service-00004-vzn
- **Status**: ✅ DEPLOYED — https://global-conflicts-athena-service-638854241992.us-central1.run.app

## Build Fix — 2026-05-29 (Revision 00005-z86)
- **Root cause**: `@deck.gl/geo-layers` transitively imports `@deck.gl/mesh-layers` 
  (via `tile-3d-layer.js`) which is not installed. Rollup hard-errored on unresolved import.
- **Fix**: Added `"@deck.gl/mesh-layers"` to `build.rollupOptions.external` in `vite.config.js`.
- **Deploy method**: Changed from `gcloud run deploy --source` to:
  1. `gcloud builds submit --config=cloudbuild.yaml` (explicit Docker build via Cloud Build)
  2. `gcloud run deploy --image=gcr.io/...` (deploy pre-built image)
  This bypasses the Cloud Run source-deploy build system which had log visibility issues.
- **Commit**: 4d8029c44f46305deed68fdd0381224b2607c807
- **Revision**: global-conflicts-athena-service-00005-z86
- **Status**: ✅ DEPLOYED — https://global-conflicts-athena-service-638854241992.us-central1.run.app

## DeckMapView Fix — CartoDB Dark Matter Basemap
- **Root cause**: Deck.gl standalone mode with `MapView` renders no basemap by default.
  Data layers (scatter/hex/arc/grid) rendered correctly but on a blank dark canvas.
- **Fix**: Added `TileLayer` + `BitmapLayer` using CartoDB Dark Matter tiles 
  (`https://basemaps.cartocdn.com/dark_all/{z}/{x}/{y}.png`) as the bottom layer.
  No API key required. Always rendered first, data layers stacked on top.
- **Commit**: 78beca0

## Data Source Replacement — 2026-05-29

### Removed
- **ACLED** — required registration/API key
- **ReliefWeb** — required pre-approved appname registration

### New Data Stack (all fully open, zero registration)
1. **UCDP GED** (`/proxy/ucdp`) — Uppsala Conflict Data, v24.1 REST API, server-side proxy
2. **GDELT** (`/proxy/gdelt`) — Real-time conflict news events, 15-min feed, 10-min server cache
3. **GDACS** (`/proxy/gdacs`) — UN Global Disaster Alert & Coordination System, real-time RSS, 15-min cache
4. **Wikidata SPARQL** (`/proxy/wikidata`) — Structured named conflicts with coordinates, 1-hour cache
5. **UNHCR Population API** (`/proxy/unhcr`) — Refugee flows + IDP data for globe arc layer, 24-hour cache
6. **RSS Direct** (`/proxy/rss`) — BBC World, Al Jazeera, AP News — server-side XML parse, no rss2json

### Globe Enhancements
- UNHCR refugee displacement arcs (animated, thickness proportional to population)
- GDACS alert pulsing rings (Red/Orange/Green severity colors)
- Legend updated with arc and ring entries

### Filter Panel
- Fully wired to store: types, sources, regions, min fatalities, date range all reactive
- Active filter count + event count shown live
- Reset button clears all filters

### Store Changes
- Replaced `acledEvents` + `reliefCrises` with `gdacsAlerts` + `wikidataConflicts` + `unhcrFlows`
- Added `regionFromCountry()` helper for automatic region classification
- Added `availableTypes`, `availableSources`, `availableRegions` computed for dynamic filter options
- `allEvents` computed now applies all filters reactively
